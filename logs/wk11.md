# Review
- items.indexOf vs items.includes: indexOf returns the specific position (index) of an element, while includes returns a boolean (true/false)
- loop patterns: similar to python

# What I learned
- When would you use a for loop vs for...of?
    - for loop: when you need the index (position) of each element, to skip elements or control step size, more control over iteration
    - for...of: when you need just the values not the index, simpler, cleaner code, to iterate through an array normally
- What does break do inside a loop? When is it useful?
    - immediately exits/stops a loop
    - useful when doing: search and exit once found, exit when condition is met, to avoid unnecessary iterations
    - some loops could be slow and wasteful without break
